Network Issues
Check connectivity, DNS, routing, and network configuration problems
▼ 5
1
Check Network Interfaces and IP Addresses
Network
Check IP Display all network interfaces and their IP addresses to verify connectivity.
ip addr show
Solution If no IP is assigned, use sudo dhclient -r && sudo dhclient to release and renew DHCP lease.
2
Test Connectivity to External Host
Network
Ping Test Test network connectivity to a known external host like Google DNS.
ping -c 4 8.8.8.8
Solution If ping fails, check firewall rules with sudo ufw status or restart network service sudo systemctl restart networking.
3
Check DNS Resolution
Network
DNS Lookup Verify DNS resolution is working properly.
nslookup google.com
Solution If DNS fails, check /etc/resolv.conf for nameservers or add nameserver 8.8.8.8 to the file.
4
Check Routing Table
Network
Route Table Display the kernel routing table to check default gateway.
ip route show
Solution If default route is missing, add it with sudo ip route add default via GATEWAY_IP.
5
Check Open Ports and Listening Services
Network
Netstat List all listening ports and associated services.
netstat -tlnp
Solution If a service isn't listening, check its status with sudo systemctl status service_name and restart if needed.

Check System Load and Uptime
Server
Load Average Check system load, uptime, and number of users.
uptime

2
Check Disk Usage
Server
Disk Space Check disk usage for all mounted filesystems.

Solution If disk is full, use du -sh /* to find large directories, then clean up or expand storage.
3
Check Memory Usage
Server
Memory Info Display memory usage statistics.
free -h
Solution If memory is low, check for memory leaks with ps aux --sort=-%mem | head or add swap space.
4
Check System Services Status
Server
Service Status Check if critical services are running.
systemctl status nginx php8.3-fpm
Solution If a service is failed, check logs with journalctl -u service_name and restart with sudo systemctl restart service_name.

▼ 3
1
Check File Permissions
Permissions
Permission Check Check permissions on web directory.
ls -la /var/www/html/
Solution Set proper permissions with sudo chown -R www-data:www-data /var/www/html and sudo chmod -R 755 /var/www/html.
2
Add User to Sudo Group
Permissions
Sudo Access Grant sudo privileges to development user.
sudo usermod -aG sudo username
Solution User needs to log out and back in for group changes to take effect.
3

Permissions
Security Modules Check if SELinux or AppArmor is blocking access.

Solution If AppArmor is enforcing, check profiles with sudo aa-status and adjust as needed.
